# Investment Risk & Control Dashboard

This repository contains an illustrative executive dashboard demonstrating how operational risk, internal audit insights, and control effectiveness can be integrated into a single view for a multi-asset investment environment.

> **Note:** This is an illustrative example using simulated data for demonstration purposes only.

---

## Overview

The dashboard is designed to provide senior leadership with clear, decision-oriented visibility across:

- Operational risk exposure across investment processes
- Control effectiveness and exception management
- Internal audit findings and systemic themes
- Risk trends impacting front-, middle-, and back-office functions

The intent is to demonstrate a practical approach to embedding governance and control oversight directly into investment workflows, rather than treating risk as a separate reporting function.

---

## Key Concepts

**Integrated Risk View**
Combines operational risk, audit findings, and control performance into a unified perspective.

**Investment Process Alignment**
Anchors risk indicators to capital markets, private investments, and financial reporting processes.

**Decision-Oriented Reporting**
Focuses on clarity, prioritization, and actionability for senior stakeholders.

**Control Transparency**
Highlights control breakdowns, emerging risks, and remediation priorities.

---

## Dashboard Sections

| # | Section | Purpose |
|---|---------|---------|
| 01 | Operational Risk Appetite Alignment | RAG-rated assessment across six investment risk domains |
| 02 | Investment Process Risk Indicators | Metric-level KRIs with trend and breach status |
| 03 | Control Effectiveness | Control testing results across trade authorization, fund accounting, and settlement |
| 04 | Audit Findings & Open Issues | Open issue register with severity, ownership, and overdue tracking |
| 05 | Counterparty & Vendor Risk | Critical relationship portfolio with highlighted exceptions |
| 06 | Emerging Risk — Investment Lifecycle | Operational incident metrics and risk function narrative |

## Technical Notes

Built as a single `index.html` file — no JavaScript libraries, CSS frameworks, or build tools. Demonstrates:

- CSS custom properties for a consistent design system
- CSS Grid for responsive two-column layout
- Semantic HTML with appropriate use of `<table>`, `<header>`, `<footer>`
- RAG status badges, progress bar segments, and stat boxes for information density
